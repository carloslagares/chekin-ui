# Chekin UI Kit

`@chekin/ui` is the shared React component library that powers every Chekin
surface: the operator Dashboard, the Guest App, embedded Host SDKs, marketing
landing pages and the onboarding flow. It replaces per-product visual
divergence with a single versioned source of truth.

## When to use

- Any new Chekin frontend screen.
- Refactors of existing surfaces — start with the Dashboard.
- Backend-built admin tools that need to look like Chekin.
- Marketing landing pages that need the brand.

## The two sources of truth

| Source | Role | URL |
|---|---|---|
| **Figma Dashboard Library** | Authoritative for *what* components exist and their intended behaviour. | `team Product Design · project 78354711` |
| **Canonical preview** | Authoritative for *how* components look. | [carloslagares.github.io/chekin-dashboard-preview](https://carloslagares.github.io/chekin-dashboard-preview/) |

When they disagree visually → **canonical wins**. When they disagree about what should exist → **Figma wins**.

## Navigating this Storybook

- **Foundations** — tokens (color, type, spacing, radii, shadows, icons).
- **Actions / Forms / Data display / Feedback / Navigation / Overlays / Surfaces / Charts** — primitives, grouped by function.
- **Layouts** — shells (AppShell, Rail, Sidebar, TopBar, AIPanel).
- **Patterns** — recurring compositions (SetupChecklist, …).
- **Examples** — full screens built entirely from kit pieces.

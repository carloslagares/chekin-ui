# Icons

Chekin uses **stroke-based** icons with a **1.5–2px** stroke. They inherit
text color through `currentColor`, so one icon file works in every tone.

## Sources

- **Lucide** is the default library — it matches Chekin's stroke profile.
- **Project-specific SVG** when Lucide doesn't have exactly the shape we
  need (e.g. the Chekin logomark, the waving-hand welcome emoji, the
  compliance shield with a check).

Don't mix in filled-icon libraries (Heroicons Solid, Material Filled) —
they clash visually with the stroke-based look.

## Conventions

- Always `stroke-linecap="round" stroke-linejoin="round"` — Chekin's hand-drawn softness.
- Use `aria-hidden="true"` when the icon is decorative (accompanies text).
- Use `aria-label="…"` when the icon carries meaning alone (a bare IconButton).
- Color via `currentColor` — never hard-code `stroke="#385BF8"`.
- Icon chips (`40×40` pale-blue circle + Main Blue icon) are Chekin's canonical "leading indicator" treatment — used on `StatCard`, `ListRow`, feature cards.

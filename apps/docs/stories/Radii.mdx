# Radii

Five canonical radii. Picking the wrong one is the single most common way
a Chekin screen loses its visual identity.

<RadiusGrid items={[
  { token: 'rounded-chekin-input',    px: 6,  name: 'Input',    use: 'Dashboard buttons and inputs.' },
  { token: 'rounded-chekin-standard', px: 8,  name: 'Standard', use: 'Inputs (large), overlays, tags.' },
  { token: 'rounded-chekin-card',     px: 14, name: 'Card',     use: 'Panels, cards, KPI tiles, timeline cards.' },
  { token: 'rounded-chekin-pill',     px: 51, name: 'Pill',     use: 'AI/conversational moments — Ask Chek, suggestion chips, marketing CTAs only.' },
  { token: 'rounded-chekin-circle',   px: 25, name: 'Circle',   use: 'Icon buttons (e.g. 40×40 overlay pickers) and avatars.' },
]} />

## Rules

- **Dashboard buttons are 6px.** Never pill-shaped inside admin UI. Pill
  radius signals AI / conversational / marketing intent — mixing them
  confuses hierarchy.
- **Cards are 14px.** Inputs/overlays inside a card are 6–8px. Never match
  the card's own radius to its contents.
- **Don't invent a 10px or 12px radius.** If something feels wrong at 8,
  the right move is usually 14, not 10.
